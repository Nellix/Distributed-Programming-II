1.2.1
Load Single Nffg
Loads the known NFFG with the given name into the remote service.
This operation can fail if an NFFG with the same name already exists in the
remote service.
Resorse:
http://localhost:8080/NffgService/rest/Nffgs/{name}
Method: POST
• Input Request
- Content-Type: application/xml
<NffgService>
<Nffgs>
<Nffg Name="Nffg1"
UpdateTime="2016-09-25T16:02:20">
<Nodes>
<Node>
</Node>
</Nodes>
<Policies>
<Policy>
</Policy>
</Policies>
</Nffg>
</Nffgs>
</NffgService>
• Output Response
- Content-Type: application/xml

1.2.2
Load both Nffg e Policy
Loads all the known NFFGs and policies to the remote service (including veri-
fication results of policies when available). This operation can fail if an NFFG
with the same name as one of the known NFFGs already exists in the remote
service. If a policy with the same name as one of the known ones already exists
in the remote service, the existing policy has to be overwritten.
Resorse:
http://localhost:8080/NffgService/rest/Nffgs/
Method: POST
•
Input Request
- Content-Type: application/xml
<NffgService>
<Nffgs>
<Nffg Name="Nffg1"
UpdateTime="2016-09-25T16:02:20">
<Nodes>
<Node>
</Node>
</Nodes>
<Policies>
<Policy>
</Policy>
</Policies>
</Nffg>
</Nffgs>
</NffgService>
•
Output Response
- Content-Type: application/xml

1.2.3
Load Reachability Policy
Loads a new reachability policy into the remote service given the policy prop-
erties. If a policy with the given name already exists in the service, the new
policy substitutes the old one. The new policy is uploaded without a verification
result.
Resorse:
http://localhost:8080/NffgService/rest/Policies/{name}
Method: POST
• Input Request
- Content-Type: application/xml
<Policies>
<ReachabilityPolicy PolicyResult="false"
name="Policy12">
<Nffg>Nffg1</Nffg>
<SrcNode name="WEBCLIENT2" />
<DstNode name="WEBSERVER7" />
</ReachabilityPolicy>
<TraversalPolicy PolicyResult="true" name="Policy18">
<Nffg>Nffg1</Nffg>
<SrcNode name="WEBSERVER3" />
<DstNode name="MAILCLIENT0" />
<TraversedFTypes>
<FType>VPN</FType>
<FType>WEB_SERVER</FType>
<FType>SPAM</FType>
</TraversedFTypes>
</TraversalPolicy>
</Policies>

1.2.4
unLoad Reachability Policy by name
Unloads the reachability policy with a given name from the remote service.
Resourse:
http://localhost:8080/NffgService/rest/Policies/{name}
Method: DELETE
• Input Request
- Content-Type: text/plain
- Example :
DELETE /NffgService/rest/Policies/Policy1 HTTP/1.1
• Output Response
- Content-Type: application/xml
- Example :
<ReachabilityPolicy PolicyResult="false"
name="Policy12">
<Nffg>Nffg1</Nffg>
<SrcNode name="WEBCLIENT2" />
<DstNode name="WEBSERVER7" />
</ReachabilityPolicy>
